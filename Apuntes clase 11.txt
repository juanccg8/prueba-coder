Cuando creamos un repositorio siempre tenemos que elegir la opción "público". 

Cuando creamos el repositorio , nos va a dar 2 listas de comandos.

El primer conjunto de comandos nos sirve cuando estamos empezando de 0 en el local (es decir, no tenemos un repositorio local). Lo copiamos y pegamos en la terminal. Es cuando estamos empezando desde 0.

En el caso en el que ya tengamos un repositorio local, selecciono el segundo repositorio. Copiamos y pegamos en la terminal y avanzamos. 

PAra hacer Git + GitHub , hay que copiar y pegar los repositorios que nos da GitHub, no hay que hacerlo con lo que está en las diapositivas. 

En el ejemplo del profe, el ya tenía la terminal creada en VSC de GIT. Entonces , lo que hizo fue a la terminal, y copió y pegó los enlaces que me dio la web.  De esa manera ya subió el repositorio, es decir, vemos la web de "index". 

El comando "git push" es un comando que sirve para subir los cambios desde la terminal de VSC a GitHub. El profe en la clase crea un commit nuevo de una carpetita de CSS y lo actualiza vía git push. 

Luego el campo "git pull" sirve para descargar los cambios integrados. En Git, el comando git pull se utiliza para fusionar los cambios remotos realizados en un repositorio en GitHub con la versión local de dicho repositorio. Realiza 2 acciones en conjunto:
	- Recupera los cambios remotos.
	- Fusiona los cambios. 

GitHub Pages --> vamos la repositorio, activamos GitHub pages. Seleccionamos la rama que queremos usar. Guardamos los cambios y GitHub cumplirá la función básica de cualquier otro hostinh.
GitHub pages solo sirve con archivos estáticos (con lso dinámicos de Java y demás no).
